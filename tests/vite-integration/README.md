# Vite Integration Test

This directory contains an example Vite + React application demonstrating how to use `@countrystatecity/countries` in a browser environment using Vite's `import.meta.glob` feature.

## What This Tests

- ✅ Using the package in a Vite browser environment
- ✅ Lazy loading of state and city data using `import.meta.glob`
- ✅ Proper code splitting and chunk generation
- ✅ TypeScript compatibility
- ✅ Build process success

## Solution Approach

This test demonstrates the community-recommended solution using Vite's `import.meta.glob` to directly access JSON files from `node_modules`, bypassing the Node.js file system dependency.

**Credits**: This approach was contributed by community members @tech-andgar and @sweethuman.

## Running the Test

```bash
# From repository root
cd tests/vite-integration

# Install dependencies
npm install

# Run dev server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

## How It Works

1. **Utility Layer** (`src/lib/countries.ts`): 
   - Uses `import.meta.glob` to map all JSON files in node_modules
   - Provides async functions that match the original package API
   - Vite automatically splits these into separate chunks

2. **Component** (`src/App.tsx`):
   - Demonstrates lazy loading countries, states, and cities
   - Shows proper loading states
   - Proves the solution works in a real React application

3. **Build Output**:
   - Initial bundle: ~5KB (just countries.json)
   - States/cities: Loaded on-demand as separate chunks
   - Total: ~4000 small chunks generated by Vite

## Key Benefits

- **No Backend Required**: Works entirely in the browser
- **Small Initial Bundle**: Only loads what's needed upfront
- **Automatic Code Splitting**: Vite handles chunking automatically
- **Type Safe**: Full TypeScript support from the original package
- **PWA Friendly**: Can be cached for offline support

## Troubleshooting

### "Cannot find module" errors

Make sure `@countrystatecity/countries` is installed in this directory:
```bash
npm install @countrystatecity/countries
```

### Paths not resolving

The glob paths must point to `/node_modules/@countrystatecity/countries/dist/data/...`. If your setup is different, adjust the paths in `src/lib/countries.ts`.

### TypeScript errors

Ensure you have the package installed with its type definitions:
```bash
npm install -D @types/node
```
